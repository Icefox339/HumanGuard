# Безопасность Интернет-приложений
## Цель

Разработка системы, выполняющей функции безопасности для Интернет-приложений, и защита ее курсовым проектом.
## Лабораторные (задачи)

У проектов есть дедлайны со своим и требованиями к результату - вехи (milestones), каждая веха - лабораторная. Если в дедлайны требования не вполняются, то выдается дополнительные индивидуальные требования к проекту.

## Порядок сдачи лабораторных
Проект должен быть реализован согласно требованиям.
По результатам выполнения проектов будут оцениваться: 
1. Сложность проекта системы.
2. Креативность реализации.
3. Качество и продуманность технической реализации.
4. Полнота документации.
5. Полнота обеспечения безопасности.
6. Количество найденных и закрытых уязвимостей.

Топ-3 команды, набравших больше всех плюсов среди всех групп, освобождаются от зачета.
