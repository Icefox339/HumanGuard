# Цель
Разработка заявленные функции системы на основе ТЗ и проекта.

# Задачи
1. Разработать перечень задач на разработку каждому члену команды.
2. Разработать функции системы и интерфейсы управления, согласно требованиям (см. `/requirements.md`).
3. Разработать часть документации к системе.
4. Разработать описание API взаимодействия с системой в нотации OpenAPI.
5. Настроить автоматизированную сборку проекта (CI).

# Требования к результату

> [!note] Инфо 
> 
>  В лабораторной указана части требований к разработке системы из полного перечня. Полный перечень требований находится в `requirements.md`. 
## 1 Функции системы
Должны быть выполнены функциональные требования к системе в соответствии с размером вашей команды:
- [ ] Аутентификация и авторизация пользователей
- [ ] Требования к управлению пользователями 
- [ ] Требования к функциям загрузки и обработки файлов
- [ ] Требования к функциям системы, согласно тематике проекта

## 2 Требования к интерфейсам управления
- [ ] Интерфейс администратора
- [ ] Внешний интерфейс HTTP REST API.

## 3 Техническая документация
- [ ] Описание системы `docs/SYSTEM_SPECS.md`.
- [ ] Рабочая документация `README.md` (в частях, помеченных комментарием "Лабораторная 2")

> [!note] Шаблоны документов
> 
> Структуры документов с примерами заполнения копировать по [ссылке](https://gitlab.com/bip_course/docs_template).

## 4 Инфраструктура
- [ ] Git-репозиторий: внедрена система версионирования согласно методике [семантического версионирования](https://semver.org/lang/ru/).
- [ ] Автоматическая сборка дистрибутивов (CI): сборка дистрибутивов (release-архивов Gitlab/Github или docker-образов) из репозитория исходных кодов.
- [ ] Запуск за реверс-прокси в виде Nginx.
