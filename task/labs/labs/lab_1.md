# Цель
Разработка ТЗ и проекта системы.

# Задачи

1. Разбиться на команды по 2-5 человек.
2. Разделить роли: бэкэнд-разработчик, девопс, аналитик, фронтенд-разработчик и тд. Обязательно должен быть 1 тимлид, который будет отвечать за всю команду.
3. Придумать тему проекта по выбранной тематике. Обязательное требование - функция ("килл-фича"), которой нет у существующих решений.
4. Завести Git-репозиторий в GitLab, завести README.md с заданной структурой (см. шаблон `README.md`)
5. Написать ТЗ в файл (см. шаблон в `docs/SRS`). 
6. Организовать трекер задач в репозитории Gitlab (группе репозиториев).
7. Направить ссылку или дать доступ на почту преподавателю.
# Требования к результату

До дедлайна необходимо согласовать с преподавателем по лабораторным:
1. Тематика проекта
2. Команда
3. Роли в команде
4. Ссылка на репозиторий(ии) Git

Те, кто до дедлайна не распределится по командам и не утвердит список выше, получат команду и тему проекта по усмотрению преподавателя.
