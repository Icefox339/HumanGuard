# Руководство пользователя

## Описание операций, доступных пользователю

---

### Операция 1: Регистрация в системе

**1. Наименование:** Регистрация нового пользователя

**2. Условия выполнения:**
- Отсутствуют сетевые ошибки доступа к системе
- Браузер поддерживает JavaScript и cookies

**3. Подготовительные действия:**
- Установить приложение-аутентификатор на мобильное устройство (Google Authenticator, Authy)

**4. Основные действия:**
- Перейти в браузере по адресу `/auth/register`
- Заполнить поле "Email"
- Заполнить поле "Password" (мин. 8 символов)
- Заполнить поле "Name"
- Нажать кнопку "Register"
- Отсканировать QR-код приложением-аутентификатором
- Сохранить резервный TOTP-секрет

**5. Заключительные действия:**
- Перенаправление на страницу входа
- Учётная запись создана

---

### Операция 2: Вход в систему

**1. Наименование:** Аутентификация пользователя

**2. Условия выполнения:**
- Пользователь зарегистрирован в системе

**3. Подготовительные действия:**
- Приложение-аутентификатор настроено

**4. Основные действия:**
- Перейти по адресу `/auth/login`
- Ввести email
- Ввести пароль
- При включённой 2FA ввести TOTP-код
- Нажать кнопку "Login"
- Или нажать "Login with Google / GitHub"

**5. Заключительные действия:**
- Перенаправление на Dashboard
- В правом верхнем углу отображается имя пользователя

---

### Операция 3: Выход из системы

**1. Наименование:** Завершение сессии

**2. Условия выполнения:**
- Пользователь авторизован

**3. Подготовительные действия:** не требуются

**4. Основные действия:**
- Нажать кнопку "Logout" в правом верхнем углу

**5. Заключительные действия:**
- Перенаправление на страницу входа
- Сессия завершена

---

### Операция 4: Смена пароля

**1. Наименование:** Изменение пароля учётной записи

**2. Условия выполнения:**
- Пользователь авторизован

**3. Подготовительные действия:** не требуются

**4. Основные действия:**
- Перейти в раздел "Profile"
- Заполнить поле "Старый пароль"
- Заполнить поле "Новый пароль" (мин. 8 символов)
- Нажать кнопку "Обновить пароль"

**5. Заключительные действия:**
- Отображение сообщения об успешной смене пароля

---

### Операция 5: Смена аватара

**1. Наименование:** Изменение аватара пользователя

**2. Условия выполнения:**
- Пользователь авторизован

**3. Подготовительные действия:** не требуются

**4. Основные действия:**
- Перейти в раздел "Profile"
- Нажать кнопку выбора файла
- Выбрать изображение (до 15 MB)
- Нажать кнопку "Сохранить"

**5. Заключительные действия:**
- Аватар обновлён в интерфейсе

---

### Операция 6: Создание API-ключа

**1. Наименование:** Генерация API-ключа для внешнего доступа

**2. Условия выполнения:**
- Пользователь авторизован

**3. Подготовительные действия:** не требуются

**4. Основные действия:**
- Перейти в раздел "API Keys"
- Заполнить поле "Название"
- Заполнить поле "Срок действия (дней)" (опционально)
- Нажать кнопку "Создать токен"
- Скопировать сгенерированный ключ (формат `hg_v1_...`)

**5. Заключительные действия:**
- Ключ добавлен в список
- Ключ используется в заголовке `X-API-Key`

---

### Операция 7: Отзыв API-ключа

**1. Наименование:** Деактивация API-ключа

**2. Условия выполнения:**
- Пользователь авторизован
- API-ключ существует

**3. Подготовительные действия:** не требуются

**4. Основные действия:**
- Перейти в раздел "API Keys"
- Найти нужный ключ в списке
- Нажать кнопку "Удалить"

**5. Заключительные действия:**
- Ключ отмечен как revoked
- Доступ по этому ключу прекращён

---

### Операция 8: Создание сайта

**1. Наименование:** Регистрация сайта для защиты

**2. Условия выполнения:**
- Пользователь авторизован

**3. Подготовительные действия:** не требуются

**4. Основные действия:**
- Перейти в раздел "Sites"
- Нажать кнопку "Добавить сайт"
- Заполнить поле "Название"
- Заполнить поле "Домен"
- Заполнить поле "Origin server"
- Заполнить поле "Настройки (JSON)" (опционально)
- Нажать кнопку "Создать сайт"

**5. Заключительные действия:**
- Сайт добавлен в список со статусом "verifying"

---

### Операция 9: Активация/приостановка защиты сайта

**1. Наименование:** Управление статусом защиты сайта

**2. Условия выполнения:**
- Пользователь авторизован
- Сайт принадлежит пользователю

**3. Подготовительные действия:** не требуются

**4. Основные действия:**
- Перейти в раздел "Sites"
- Найти нужный сайт
- Нажать кнопку "Activate" или "Suspend"

**5. Заключительные действия:**
- Статус сайта изменён
- При статусе "active" запросы к `/api/check` обрабатываются

---

### Операция 10: Удаление сайта

**1. Наименование:** Удаление сайта из системы

**2. Условия выполнения:**
- Пользователь авторизован
- Сайт принадлежит пользователю

**3. Подготовительные действия:** не требуются

**4. Основные действия:**
- Перейти в раздел "Sites"
- Найти нужный сайт
- Нажать кнопку "Delete"
- Подтвердить удаление

**5. Заключительные действия:**
- Сайт удалён
- Связанные сессии и блэклист удалены

---

### Операция 11: Просмотр и изменение настроек защиты

**1. Наименование:** Настройка параметров детекции ботов

**2. Условия выполнения:**
- Пользователь авторизован
- Сайт принадлежит пользователю

**3. Подготовительные действия:** не требуются

**4. Основные действия:**
- Перейти в раздел "Sites"
- Нажать ссылку "Settings" у нужного сайта
- Просмотреть текущие настройки в JSON-формате
- Отредактировать JSON
- Нажать кнопку "Сохранить настройки"

**5. Заключительные действия:**
- Настройки применены
- Детектор использует новые параметры

---

### Операция 12: Управление блэклистом IP-адресов

**1. Наименование:** Блокировка IP-адресов для сайта

**2. Условия выполнения:**
- Пользователь авторизован
- Сайт принадлежит пользователю

**3. Подготовительные действия:** не требуются

**4. Основные действия (добавление):**
- Перейти в раздел "Sites"
- В карточке сайта найти секцию "Blacklist"
- Ввести IP-адрес (формат IPv4)
- Ввести причину блокировки (опционально)
- Нажать кнопку "Добавить IP"

**5. Основные действия (удаление):**
- Найти IP в списке черного списка
- Нажать кнопку "Удалить"

**6. Заключительные действия:**
- Запросы с заблокированного IP получают ответ 403

---

### Операция 13: Загрузка файла

**1. Наименование:** Загрузка файла в систему

**2. Условия выполнения:**
- Пользователь авторизован

**3. Подготовительные действия:** не требуются

**4. Основные действия:**
- Перейти в раздел "Files"
- Нажать кнопку выбора файла
- Выбрать файл (до 5 ГБ)
- Нажать кнопку "Загрузить файл"
- Отслеживать прогресс загрузки

**5. Заключительные действия:**
- Файл отображается в списке
- Доступен для скачивания и шаринга

---

- Пользователь авторизован

## Операции, доступные только администратору

---

### Операция A1: Просмотр списка пользователей

**1. Наименование:** Просмотр всех зарегистрированных пользователей

**2. Условия выполнения:**
- Пользователь авторизован с ролью "admin"

**3. Подготовительные действия:** не требуются

**4. Основные действия:**
- Перейти в раздел "Admin" → "Users"

**5. Заключительные действия:**
- Отображена таблица с email, именем, ролью, датой регистрации

---

### Операция A2: Изменение роли пользователя

**1. Наименование:** Назначение/изменение прав пользователя

**2. Условия выполнения:**
- Пользователь авторизован с ролью "admin"

**3. Подготовительные действия:** не требуются

**4. Основные действия:**
- Перейти в раздел "Admin" → "Users"
- Выбрать пользователя
- Изменить значение в поле "Role" (user/admin)
- Нажать "Сохранить"

**5. Заключительные действия:**
- Роль пользователя обновлена

---

### Операция A3: Деактивация сессии пользователя

**1. Наименование:** Принудительное завершение сессии пользователя

**2. Условия выполнения:**
- Пользователь авторизован с ролью "admin"

**3. Подготовительные действия:** не требуются

**4. Основные действия:**
- Перейти в раздел "Admin" → "User Sessions"
- Найти нужную сессию
- Нажать кнопку "Revoke"

**5. Заключительные действия:**
- Сессия удалена
- Пользователь вынужден войти заново
